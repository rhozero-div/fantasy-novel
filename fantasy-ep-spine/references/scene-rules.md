## Scene 设计规则

---

## Scene 定义

**Scene = 剧情走向的一个自然段落。**

Scene 的边界跟着事件走：一个完整的起承转合 = 一个 Scene。Scene 完成后，叙事节奏进入自然停顿点，读者可以停在这里不追问「然后呢」。

### Scene 判定标准

问：**这个段落是否有完整的叙事节奏（起→承→转→合）？**

- 是 → 一个独立的 Scene
- 否 → 合并进相邻 Scene

### Scene 边界

Scene 边界出现在：
- 叙事节奏进入自然停顿点（上一事件流已结束，下一事件流尚未开始）
- 场景/地点切换（但切换必须有叙事理由）
- POV 切换（但切换必须服务于事件流）

### 物理事件禁止跨 Scene 拆分

战斗、追逐、危机等物理事件流必须在同一个 Scene 内完成（以危机解除或事件流结束结尾）。

### 弧光定位

- **EP 级弧光**：用户点火时设定，是 EP 的整体位移目标
- **Scene 级弧光**：Scene 负责推进 EP 弧光，不做独立弧光要求
- **Scene 拆分依据**：事件流的自然分段，不是弧光节点

---

## Scene 拆分流程

1. 列出本 EP 内的事件流（起承转合）
2. 按「叙事节奏的自然停顿点」分段
3. 每段 = 一个 Scene
4. 验证：每段是否有完整的叙事节奏？读者能在这里停顿不追问「然后呢」？

---

## 典型 Scene 结构

### 文戏为主 Scene

```
Scene X：谈判/对峙
  └─ 完整起承转合：立场试探 → 底线博弈 → 破局或崩溃
```

### 武戏为主 Scene

```
Scene Y：战斗
  └─ 完整战斗流程：遭遇与试探 → 战术调整与逆转 → 终结与结果
```

### 混合 Scene

```
Scene Z：战斗余波 + 关系重建
  └─ 战后清点 → 角色对话，弧光落地
```
